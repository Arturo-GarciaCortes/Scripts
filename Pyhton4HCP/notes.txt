--- Threads vs Processes ---

- For a clearer explanation about threads and processes see the jupyter notebook: threads_vs_processes
- Threads allow for multi-execution
- Processes only one. However, it is also to do multi-processes.

- The purpose of multi-threading is to leverage all the cores available for the calculation. Each thread can run in parallel --> Multi-threading. 
- Python in particular has the problem of global interpreter loc --> All the threads wants the loc, and only one is using it, so all the threads are awaiting for thread 1 to release it so the others can do their job. 
- Nevertheless, some libraries release the global interpreter loc (f.eg. numpy). In this case a multi-thread approach will perform much better than a processes computation. 
- Doing computations with processes can be beneficial in some tasks. 

the dask.compute() function allows for choosing between using threads or processes when performing a task in python, as showed in the jupyter notebook: threads_vs_processes part 4. CPU-Bound Task and part 5. I/O Bound Task.


--- Numba Basics library ---

- It compiles our python functions and speeds them up inside the python code: Uses C++ tools to transform our python functions into machine code-compiled functions, making them more efficient (as if we're using C++) --> It gives a C++ speed to Python functions.

- Speaker recommends to use Numba to optimize only the computationally heavy tasks within our python code.

- Namba is called by only putting @jit up where the function is defined (See 0_basics.ipynb). 

- We can do @jit on functions that inside use already optimized (@jit) functions. 

- Numba also reduces the memory usage compared to a python function. 

--- Multithreading with Numba ---

- The goal is to release the global interpreter loc. But Numba can also do multithreading for us (run in parallel).

- To parallelize with Numba do @jit(parallel=True) up to where the function is defined (see 2_threads.ipynb). 

If we wanna use all the cores

OMP_NUM_THREADS --> it specifies the number of cores that we are gonna use to make parallel calculations. Typically, if the parallelization in our code is of 100%, this flag is set as the total number of cores requested and available for the calculation.
